プロジェクト作成

フォルダ構造を作成し、設定を保存します。

gs new # または gs prepare <name> でフォルダのみ作成
作成されるフォルダ:
images_S/ (全天球), images_P/ (透視投影+XMP), Spherical/ (Metashape), RS/ (RealityScan), tools/, output/, colmap/
project/ ├─ images_S/ ← 全天球画像 ├─ images_P/ ← 透視投影画像 + XMP ├─ Spherical/ ← Metashape出力 (camera.xml) ├─ RS/ ← RealityScanプロジェクト ├─ tools/ ← バッチスクリプト └─ output/ ← 最終出力

フレーム抽出

360度動画から全天球フレームを images_S/ に抽出します。

上のボタンでコマンドを生成

Metashapeアライメント

全天球画像をMetashapeでアライメントし、camera.xmlをエクスポートします。

手順:
1. Metashape → Workflow → Add Photos → images_S/ を追加
2. Camera Type: Spherical に設定
3. Workflow → Align Photos
4. File → Export → Export Cameras → Spherical/camera.xml
座標系: Metashapeは Z-up / c2w / OpenCVカメラ座標系。
XMP生成時にRSの Y-up / OpenGL座標系に自動変換されます。

FFmpeg分割

全天球画像を複数方向の透視投影画像に分割します。

上のボタンでコマンドを生成
命名規則: {フレーム番号}_{方向角度}.jpg
例: 0001_000.jpg, 0001_045.jpg, ... 0001_315.jpg
CLIからも実行可: gs split <name> -i <video>

XMP生成

Metashapeのcamera.xmlから各透視投影画像のXMPサイドカーファイルを生成します。

座標変換 (Pattern I) 修正済み:
回転: R_rs = [ R[0], −R[2], R[1] ]  (Z-up c2w → Y-up c2w + カメラフレーム変換)
位置: (x, y, z) → (x, −z, y)
RSの正解データと照合済み (roll誤差 < 0.8度)
上のボタンでコマンドを生成
座標変換の詳細
Metashape座標系: Z-up, c2w (camera-to-world), OpenCVカメラ (X右, Y下, Z前方)
RS座標系: Y-up, c2w, OpenGL-likeカメラ (X右, Y上, Z後方)

変換行列 S': [[1,0,0], [0,0,-1], [0,1,0]]
1. c2w回転にyaw回転を適用: R = R_c2w @ R_yaw(angle)
2. row0 = R[0] (そのまま)
3. row1 = -R[2] (Z-upのZ軸反転)
4. row2 = R[1] (Z-upのY軸)

RealityScanアライメント

XMP付き画像をRSに読み込み、exactモードでアライメントします。

手順:
1. RS → New Project → images_P/ を読み込み (XMP自動検出)
2. Align Settings → Use Camera Metadata: Exact
3. Align Photos 実行
4. 8方向が正しく展開されるか確認
5. Registration (.csv) + Point Cloud (.ply) エクスポート
画像が多い場合:
4分割してRSで個別Align → 後でMergeすると効率的です。

3DGSトレーニング

RSのアライメント結果を使って3D Gaussian Splattingを実行します。

RSからエクスポートした Registration (.csv) と Point Cloud (.ply) を
Postshot や gsplat などに渡してトレーニング。

回転診断ツール

XMPの座標変換が正しいか診断するスクリプト。10パターンの変換を試し、RSの正解データと比較します。

cd F:\3DGS python tools/diagnose_rotation.py
結果: Pattern I (c2w + 行1,2反転) がRSの正解パターンと一致。
roll差分誤差: < 0.1度 / 絶対roll誤差: < 0.8度

テスト済みパターン

パターン 説明 roll誤差
Ac2w + OpenCV (旧スクリプト)90.0度
Bc2w + OpenGL (列反転)90.0度
Ec2w + OpenCV + 負yaw0.07度
I c2w + 行1,2反転 (採用) 0.06度

Pattern I は絶対rollも一致 (EはΔrollのみ一致、絶対値は90度ずれ)
